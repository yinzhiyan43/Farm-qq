## 一、Bug修复记录（2026-05-05）

### 1. ✅ 全局背景变黑（farm_bg01 丢失）

**原因**：上次优化将 farm_bg01 从场景引用改为异步 placeholder，null spriteFrame + CUSTOM 模式渲染为 1×1 像素导致全黑。

**修复**：恢复 farm_bg01 的 `_spriteFrame` UUID 引用到 MainScene.scene，移除 GameManager 中的异步加载代码。全局背景不应延迟加载。

### 2. ✅ 仓库/商店面板背景透明

**原因**：`UISimplePanel.makeNode()` 中 `CUSTOM` 模式下 null spriteFrame 以原始纹理尺寸(1×1)渲染。

**修复**：改为 `TRIMMED` 模式——使用 UITransform 的 contentSize 渲染，内置白色纹理 + Color tint 正确显示墨绿色背景。

### 3. ✅ 点击牌子触发 fireworks 动画加载错误

**原因**：清空了 fireworks Animation 组件的 `_clips`，`resources.load('animation/fireworks')` 找不到资源。

**修复**：
- 恢复 fireworks Animation 组件的 `_clips` 和 `_defaultClip`
- 修正 cloud 节点错误覆盖的 Animation 组件
- brand.ts 异步加载代码保留作为容灾

### 4. ✅ bird 动画路径修正

**原因**：bird.anim 在 `assets/animation/`（非 resources），但 GameManager 使用 `resources.load('animation/bird')` 加载。

**修复**：bird.anim 移动到 `assets/resources/animation/bird.anim`。

### 5. ✅ 启动性能优化

| 问题 | 改进 | 文件 |
|------|------|------|
| 游戏启动时立即加载烟花和飞鸟预制体 | 烟花按需加载（首次点击播放才加载） | Brand.ts |
| 飞鸟动画在启动加载阻塞UI | 飞鸟异步加载（游戏启动完成后再加载） | Brand.ts |
| `brandEnter()` 500ms硬编码延时 | 拆分 `brandLogo()` + `brandMain()`，独立回调 | Brand.ts |

### 6. ✅ 商店首次打开白屏

**原因**：数据异步加载未完成时打开商店。

**修复**：异步重试机制 — 检测 `shopManager.isDataReady()`，0.15s 间隔重试×10次。

### 7. ✅ Tab切换Bug

**原因**：切换Tab时未重置 `_shopRetryCount`。

**修复**：切换Tab时重置重试计数。默认Tab改为"已解锁"。

## 二、代码审查报告

### 🔴 高优先级

#### 1. localStorage 性能热点

**文件**: `AchievementManager.ts`, `TaskManager.ts`, `ExpSystem.ts`

**问题**: 每次 `updateProgress()` 都同步执行 `localStorage.setItem()`。游戏中高频事件（连续收获）导致每帧多次同步写入，严重影响帧率。

**建议修复**:
```typescript
// 延迟合并写入
private _dirty: boolean = false;
private _markDirty() {
    if (!this._dirty) {
        this._dirty = true;
        this.scheduleOnce(() => this._flushSave(), 2.0);
    }
}
```

#### 2. AnimationSystem 未使用对象池

**文件**: `AnimationSystem.ts` vs `PerformanceSystem.ts`

**问题**:
- `playCoinFlyEffect()` / `playMatureIndicator()` 每次创建新 Node
- `PerformanceSystem` 已实现三个对象池但 AnimationSystem 完全没用
- 高频收获时频繁创建/销毁节点触发GC

**建议**: AnimationSystem 改为从 `PerformanceSystem.getFromPool()` 获取节点。

#### 3. EventBus 双重事件体系不一致

**问题**:
- 部分 Manager 同时使用本地 `EventTarget` + `EventBus` 全局事件
- TutorialManager 用 `this.node.emit()` 而不是 EventBus

**建议**: 统一使用 EventBus，移除各 Manager 的私有 `_eventTarget`。

#### 4. 存档数据收集依赖场景查找

**文件**: `SaveManager.ts:433`
```typescript
const soil = this.node.scene.getComponentInChildren(Soil);
```
- 依赖节点树结构，耦合度高
- `getComponentInChildren` 在大型场景中开销较大
